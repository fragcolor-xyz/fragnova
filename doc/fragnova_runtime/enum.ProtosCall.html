Expand description

Contains one variant per dispatchable that can be called by an extrinsic.

upload

Fields

§references: Vec<[u8; 32], Global>
§category: Categories
§tags: BoundedVec<BoundedVec<u8, <T as Config>::StringLimit>, <T as Config>::MaxTags>
§linked_asset: Option<LinkedAsset>

§cluster: Option<[u8; 16]>
§data: ProtoData

Upload a Proto-Fragment onto the Blockchain. Furthermore, this function also indexes data in the Blockchain’s Database and makes it available via bitswap (IPFS) directly from every chain node permanently.

Arguments

  • origin - The origin of the extrinsic function
  • references - List of other Proto-Fragments used to create the Proto-Fragment
  • categories - Category type of the Proto-Fragment
  • tags - List of tags to tag the Proto-Fragment with
  • linked_asset (optional) - An asset that is linked with the Proto-Fragment (e.g an ERC-721 Contract)
  • license - Enum indicating how the Proto-Fragment can be used. NOTE: If None, the Proto-Fragment can’t be included into other Proto-Fragments
  • cluster - the Cluster id the proto belongs to (Optional)
  • data - Data of the Proto-Fragment

patch

§proto_hash: [u8; 32]
§license: Option<UsageLicense<<T as Config>::AccountId>>
§new_references: Vec<[u8; 32], Global>
§tags: Option<BoundedVec<BoundedVec<u8, <T as Config>::StringLimit>, <T as Config>::MaxTags>>
§data: Option<ProtoData>

Patch an existing Proto-Fragment (by appending the hash of data to the Vector field patches of the existing Proto-Fragment’s Struct Instance) Furthermore, this function also indexes data in the Blockchain’s Database and stores it in the IPFS

Arguments

  • origin - The origin of the extrinsic function
  • proto_hash - Existing Proto-Fragment’s hash
  • license (optional) - If this value is not None, the existing Proto-Fragment’s current license is overwritten to this value
  • new_references - List of New Proto-Fragments that was used to create the patch (data needs to be populated, or it has no effect)
  • tags (optional) - List of tags to overwrite the Proto-Fragment’s current list of tags with, if not None.
  • data - Data of the Proto-Fragment

set_metadata

Fields

§proto_hash: [u8; 32]
§metadata_key: BoundedVec<u8, <T as Config>::StringLimit>
§data: Vec<u8, Global>

Alters the metadata of a Proto-Fragment (whose hash is proto_hash) by adding or modifying a key-value pair (metadata_key.clone,blake2_256(&data.encode())) to the BTreeMap field metadata of the existing Proto-Fragment’s Struct Instance. Furthermore, this function also indexes data in the Blockchain’s Database and stores it in the IPFS To successfully patch a Proto-Fragment, the auth provided must be valid. Otherwise, an error is returned

Arguments

  • origin - The origin of the extrinsic / dispatchable function
  • proto_hash - The hash of the Proto-Fragment
  • metadata_key - The key (of the key-value pair) that is added in the BTreeMap field metadata of the existing Proto-Fragment’s Struct Instance
  • data - The hash of data is used as the value (of the key-value pair) that is added in the BTreeMap field metadata of the existing Proto-Fragment’s Struct Instance

detach

Fields

§proto_hashes: Vec<[u8; 32], Global>
§target_chain: SupportedChains
§target_account: BoundedVec<u8, <T as Config>::DetachAccountLimit>

Request to detach a Proto-Fragment from Fragnova.

Note: The Proto-Fragment may actually get detached after one or more Fragnova blocks since when this extrinsic is called.

Note: Once the Proto-Fragment is detached, an event is emitted that includes a signature. This signature can then be used to attach the Proto-Fragment to an External Blockchain target_chain.

Arguments

  • origin - The origin of the extrinsic function
  • proto_hashes - IDs of the Proto-Fragments to detach
  • target_chain - External Blockchain to attach the Proto-Fragment into
  • target_account - Public Account Address in the External Blockchain target_chain to assign ownership of the Proto-Fragment to
